### Основные флаги `pytest`

Эти флаги позволяют управлять тем, какие тесты запускать и как отображать результат.

| Флаг (короткий) | Флаг (длинный) | Описание | Пример использования |
| :--- | :--- | :--- | :--- |
| `-v` | `--verbose` | Включает "многословный" режим. Выводит более подробную информацию о каждом запущенном тесте (PASSED, FAILED, SKIPPED). | `pytest -v` |
| `-k` | `--keyword` | Запускает тесты, имена которых содержат определенное ключевое слово или выражение. Можно использовать `and`, `or`, `not`. | `pytest -k "login and not registration"` |
| `-m` | `--mark` | Запускает тесты, помеченные определенной меткой (маркером). Маркеры задаются в коде с помощью декоратора `@pytest.mark.имя_маркера`. | `pytest -m regression` |
| `-s` | `--capture=no` | По умолчанию `pytest` "перехватывает" и скрывает все, что выводит ваш код (например, через `print()`). Этот флаг отключает перехват, и все выводы в консоль будут видны. | `pytest -s` |
| `-x` | `--exitfirst` | Немедленно останавливает выполнение тестов после первой же ошибки (первого упавшего теста). | `pytest -x` |
| `--tb` | `--traceback` | Управляет стилем вывода трейсбека (отчета об ошибке). Например, `--tb=line` покажет только одну строку для каждой ошибки. | `pytest --tb=short` |
| `--maxfail=NUM` | | Останавливает выполнение тестов после указанного количества `NUM` ошибок. | `pytest --maxfail=3` |
| `-h` | `--help` | Показывает все доступные флаги и опции командной строки. | `pytest -h` |

### Запуск конкретного теста

Вы также можете запустить один конкретный файл или даже один тест внутри файла, указав путь к нему.

*   **Запустить все тесты в файле:**
    ```bash
    pytest tests/test_dashboard.py
    ```
*   **Запустить один конкретный тест в файle:**
    ```bash
    pytest tests/test_dashboard.py::test_dashboard_displaying
    ```
